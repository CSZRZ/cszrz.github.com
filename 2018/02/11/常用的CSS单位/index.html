    • 比较常用的 css 单位的介绍，主要是 pt，px，%，em，rem，vh，vw，当然还有些其他的，暂且不表。

    

    pt

    pt(point) 是印刷行业的常用单位，等于 1/72 英寸，表示绝对长度

    px

    px(piexl) 指的是像素，屏幕上显示数据的最小单位，表示相对单位，相对屏幕的分辨率。

    %

    百分比，相对于父元素的字体大小

    em

    相对单位，相对于当前对象内文本的字体尺寸，即 em 的计算是基于父级元素 font-size 的，所以如果一个嵌套结构的情况下，每个元素大小都是相对于父元素的，所以需要进行一层层的计算。 e.g.

    rem

    相对单位，root em ，相对于根元素（HTML）字体大小。e.g.

    vw

    相对单位 1/100的视口的宽度，响应式网页设计技术很大程度上依赖于比例规则，然而 % 比例不是总是合适的，因为 CSS 宽度是相对于最近的包含父元素的，如果我们的参考系是是显示窗口而不是父元素呢？vh,vw 应运而生，100vh，就代表了占满整个屏幕高度，方便我们实现基于窗口为参考系的实现。

    vh

    相对单位 1/100的视口的高度

    vmin

    相对单位 1/100的视口的小尺寸（高度或宽度）

    vmax

    相对单位 1/100的视口的大尺寸（高度或宽度）

    小结

     px 和 em 应该还是 PC 端实现的常用单位，对于我个人而言，关注的是移动端，所以 rem 和 vw,vh 是比较常用的，比较好做适配，这里有个 trick 记录下，默认情况下，浏览器的字体大小为 16px，这时设置根元素字体大小为 62.5%， 这时候 1rem = 10px(16px x 62.5% = 10px),方便我们后续单位的换算.
