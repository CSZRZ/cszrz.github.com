Reactive Native 是 facebook 开源的一套跨平台方案，主要用于 iOS&Android 双端使用，适合移动端和前端的人员学习使用，但对于各自的学习曲线是不一样的。学习 Reactive Native 需要学习 React，JS，css 等一系列前端的技术栈，所以前期对于客户端人员来说就是学习上手前端的知识，但是 Reactive Native 用的是原生的组件，所以对于前端人员来说需要了解移动端的生态，总而来说是各有利弊的，但是就目前来看，大前端这一概念想必已经是业界公认了，随着技术的发展，程序员们更加不能局限一门语言，一门技术，空余时间学习了解下其他方向的，移动端，前端，后端之前相互学习了解，可以开阔自己的思路，并且在平时和其他端同事对接时沟通更加顺畅，当然这只是一个美好的愿望，毕竟人的精力是有限的，知识的深度和广度也需要自己去衡量，随着年纪的上涨，家庭在平时的占比也会越来越大，这些暂且不表，现在能学一点是一点嘛！！

The Basics

支持 ES2015 语法。import, from, class, extends, 和 () => 等。JSX: Hello world!. 一种语法将 xml 嵌入到 JS 中。还有就是组件的概念，很多我们看到的界面上的东西都是一个组件构成的，一个简单的组件需要一个 render 函数，返回一些 JSX 去渲染、

组件

组件里面有几个比较重要的概念、

  • props（属性）:

{} 的意思是括号内部是一个js变量或者表达式，需要执行后取值

  • state（状态）

props 和 state 是控制一个组件的 2 种数据类型。属性是 parent 设置的，并且在组件的整个生命周期中是固定的，如果数据需要改变，我们需要使用 state。通常需要在构造器中去初始化 state，然后需要改变的时候调用 setState

样式

不需要特定的语言或者语法去定义样式，你只需要使用 JS。所有的核心组件都接受名为 style 的样式。这些样式名基本上是遵循了web上的CSS的命名，只是按照JS的语法要求使用了驼峰命名法，例如将 background-color 改为 backgroundColor。style 属性可以是一个普通的JavaScript对象。这是最简单的用法，还可以传入一个数组——在数组中位置居后的样式对象比居前的优先级更高，这样你可以间接实现样式的继承。际开发中组件的样式会越来越复杂，我们建议使用StyleSheet.create来集中定义组件的样式。比如：

宽高

  • 指定宽高，最简单的给组件设定尺寸的方式就是在样式中指定固定的 width 和 height。RN 中的尺寸都是无单位的，代表像素无关密度。

这样给组件设置尺寸也是一种常见的模式，比如要求在不同尺寸的屏幕上都显示成一样的大小

  • 弹性宽高

在组件样式中使用flex可以使其在可利用的空间中动态地扩张或收缩。一般而言我们会使用flex:1来指定某个组件扩张以撑满所有剩余的空间。如果有多个并列的子组件使用了flex:1，则这些子组件会平分父容器中剩余的空间。如果这些并列的子组件的flex值不一样，则谁的值更大，谁占据剩余空间的比例就更大（即占据剩余空间的比等于并列组件间flex值的比）。

使用 flexbox 布局

RN 中 flexbox 的工作原理和 web 上的 css 基本一致，但是也存在差异。flexDirection的 默认值是 column 而不是 row，而 flex 也只能指定一个数字值。

Flex Direction

在组件的 style 中指定 flexDirection 可以决定布局的主轴，默认是竖直轴(column)

Justify Content

在组件的style中指定justifyContent可以决定其子元素沿着主轴的排列方式。子元素是应该靠近主轴的起始端还是末尾段分布呢？亦或应该均匀分布？对应的这些可选项有：flex-start、center、flex-end、space-around以及space-between。

Align Items

在组件的style中指定alignItems可以决定其子元素沿着次轴（与主轴垂直的轴，比如若主轴方向为row，则次轴方向为column）的排列方式。子元素是应该靠近次轴的起始端还是末尾段分布呢？亦或应该均匀分布？对应的这些可选项有：flex-start、center、flex-end以及stretch。

ScrollView

ScrollView适合用来显示数量不多的滚动元素。放置在ScollView中的所有组件都会被渲染，哪怕有些组件因为内容太长被挤出了屏幕外。如果你需要显示较长的滚动列表，那么应该使用功能差不多但性能更好的ListView组件

ListView

React Native提供了几个适用于展示长列表数据的组件，一般而言我们会选用FlatList或是SectionList。

FlatList

FlatList组件用于显示一个垂直的滚动列表，其中的元素之间结构近似而仅数据不同。

FlatList更适于长列表数据，且元素个数可以增删。和ScrollView不同的是，FlatList并不立即渲染所有元素，而是优先渲染屏幕上可见的元素。

FlatList组件必须的两个属性是data和renderItem。data是列表的数据源，而renderItem则从数据源中逐个解析数据，然后返回一个设定好格式的组件来渲染。

SectionList

SectionList 有点类似于UITableView的group样式,带有section header。
