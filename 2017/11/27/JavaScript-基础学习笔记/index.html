JavaScript 最早是由网景公司开发的一款脚本语言，原来叫 LiveScript ,然后为了蹭 Java 的热度改名叫做 JavaScript。所以对于刚接触编程的人来说，听到这 2 个的名字总让人困惑以为两者之前有什么关系，其实上就是一个“碰瓷”的…后来微软发布了它们的 JScript 脚本语言，再之后，网景公司就把 JavaScript 提交到 Ecma International——一个欧洲标准组织（计算机发展的过程中，涉及到很多标准化的制定，没有标准的世界是无序的，添加了很多额外的成本，比如 RFC，UTF-8）。在 1999 制定了 ECMAScript 3 标准，。这版标准直到现在都是十分牢固的。第 4 版的制定废弃了，由于对于语言复杂程度的不同阵营的分歧，2009 年发布了 ECMAScript 5，这版很多基础的形成来源于废弃的第 4 版。然后在 2015 年发布了 ESMAScript 6. JS 不像其他编程语言有输入输出的概念，它设计的初衷就是跑在一个宿主环境的脚本语言，依靠宿主提供的机制和外部沟通。

概述

JS 是一门动态语言，语法受到 Java 和 C 影响, JS 支持面向对象编程，通过对象原型来替代 classes，JS 同样也支持函数编程，函数也是对象 ，函数可以执行代码，也能在对象之间传递。下面就从 JS 中的类型来开始介绍：

  • Number
  • String
  • Boolean
  • Symbol(new in ES2015)
  • Object
    • Function
    • Array
    • Date
    • RegExp
  • null
  • undefined

Other types

JavaScript 中 null 和 undefined 是不同的，前者表示一个空值（non-value），必须使用 null 关键字才能访问，后者是“undefined（未定义）”类型的对象，表示一个未初始化的值，也就是还没有被分配的值.undefined 实际上是一个不允许修改的常量

JS 有一个布尔值类型，（true/false）,任何值都可以按照下面规则转换成一个布尔值:

  • false、0、空字符串("")、NaN、null 和 undefined 被转换为 false
  • 所有其他值被转换为 true

JavaScript 与其他语言的（如 Java）的重要区别是在 JavaScript 中语句块（blocks）是没有作用域的，只有函数有作用域。因此如果在一个复合语句中（如 if 控制结构中）使用 var 声明一个变量，那么它的作用域是整个函数（复合语句在函数中）。 但是从 ECMAScript Edition 6 开始将有所不同的， let 和 const 关键字允许你创建块作用域的变量。
