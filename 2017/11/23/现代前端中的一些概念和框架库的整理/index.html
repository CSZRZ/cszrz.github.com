工具类

npm

npm 是前端广泛使用的包管理工具，通过 npm 可以让我们方便的发布和使用别人封装的库，其实就类似 CocoaPods。CocoaPods 通过生成一个 Podfile 文件来配置我们使用的第 3 库信息，npm 则是通过一个 package.json 文件来配置我们的引用信息.

创建 package.json

一般在项目根目录下执行 npm init 或者通过 npm init –yes 直接生产一个默认值的 package.json

package.json 的内容

基本上看到就能明白什么意思了，name 和 version 是必须要有的 2 部分，下面就是项目的描述，作者，支持的脚本，之类的，还有 main 表示入口文件，一把都是 index.js，下面说一下项目的依赖，就像 pod install 一样，执行 npm install 根据配置的依赖去下载相关的库、包有 2 种依赖方式，在生产环境中使用的 dependencies 和 在开发测试环境中使用的 devDependencies

语义化版本规则：

正常我们做项目涉及到版本的升级迭代，一般都是 3 位数，从 1.0.0 版本开始，第 1 位表示大版本更新，第 2 位表示小版本迭代，最后一位一般都是修复 bug。npm 这里发布版本的时候，第 2 位版本发布的时候表示不会影响之前的版本，大版本可能无法兼容之前的，所以我们一般都会指定第 2 种形式，允许更新小版本.

作为使用者，我们可以在 package.json 文件中写明我们可以接受这个包的更新程度（假设当前依赖的是 1.0.4 版本）：

  • 如果只打算接受补丁版本的更新（也就是最后一位的改变），就可以这么写：
    1.0
    1.0.x
    ~1.0.4

  • 如果接受小版本的更新（第二位的改变），就可以这么写：
    1
    1.x
    ^1.0.4

  • 如果可以接受大版本的更新（自然接受小版本和补丁版本的改变），就可以这么写：
    *
    x

安装 package

安装 package 有 2 种方式，一种本地（当前项目路径）安装，另一种是全局安装，npm install 默认是安装本地的，在当前项目通过 require（）加载使用。npm install 默认会安装 所有依赖，包括正式的和开发环境中的，如果只想安装正式的，需要执行 npm install --production

  • 安装某一个 package

npm install <package_name> 这个命令会在当前目录下创建一个 node_modules 目录，然后将包下载到这个目录中。

  • 安装指定版本
  • 安装参数 –save 和 –save-dev

npm install <package_name> --save 表示将这个包名及对应的版本添加到 package.json的 dependencies

npm install <package_name> --save-dev 表示将这个包名及对应的版本添加到 package.json的 devDependencies

更新本地 package

使用 npm outdated 查看 package 是否更新，使用 npm update <package-name> 更新单个，使用 npm update 更新所有

卸载本地 package

npm uninstall <package-name>

npm run

npm 还可以直接运行 package.json 中 scripts 指定的脚本:

npm run 是 npm run-script 的缩写。命令行输入 npm run lint 或者 npm run-script lint 就会执行 jshint **.js

npm run 会创建一个Shell，执行指定的命令，并临时将node_modules/.bin加入PATH 变量，这意味着本地模块可以直接运行。

package.json 中的 scripts 执行的脚本是本地项目内 node_modules -> .bin 内的脚本

webpack

webpack 是一个现代 JavaScript 应用程序的模块打包器(module bundler)。当 webpack 处理应用程序时，它会递归地构建一个依赖关系图(dependency graph)，其中包含应用程序需要的每个模块，然后将所有这些模块打包成一个或多个 bundle。

预处理

less

Less（或写作LESS）是一种由Alexis Sellier设计的动态层叠样式表语言，受Sass所影响，同时也影响了Sass的新语法：SCSS。[1]
LESS是开源的，其第一个版本由Ruby写成，但在后续的版本当中，Ruby逐渐被替换为JavaScript。受益于JavaScript，LESS可以在客户端上运行（IE6+、Webkit、Firefox），也可以在服务端运行（Node.js、Rhino）。[2]
在语法方面，LESS与CSS较为接近，一个合法的CSS代码段本身也是一段合法的LESS代码段。LESS提供变量、嵌套、混合、操作符、函数等一般编程所需的抽象机制.
